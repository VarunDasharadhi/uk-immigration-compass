# Dark Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a manual sun/moon theme toggle to the header that defaults to the visitor's OS preference, with a matching dark theme applied consistently across every page, including gradients, glows, shadows, and the one non-Tailwind chart (petitions bar chart uses hardcoded hex colors via Recharts props, not CSS classes).

**Architecture:** Tailwind `darkMode: 'class'` strategy driven by a `ThemeContext` (`contexts/ThemeContext.tsx`) backed by `localStorage`, with an inline script in `index.html` that sets the `dark` class on `<html>` before React mounts (no flash of the wrong theme). Every component gets paired `dark:` Tailwind variants; the one place color isn't a CSS class (the Recharts bar chart in `PetitionTracker.tsx`) reads `theme` from `useTheme()` and picks a small literal color palette.

**Tech Stack:** React 18, TypeScript, Tailwind CSS 3, Vite, Jest + Testing Library (already configured, no new dependencies needed — `Sun`/`Moon` icons already exist in the installed `lucide-react`).

## Global Constraints

- No new npm dependencies — everything needed (`darkMode: 'class'`, `lucide-react` icons, `localStorage`, `matchMedia`) is already available.
- Every task must pass `npx tsc --noEmit` before being considered done.
- Follow the spec at `docs/superpowers/specs/2026-07-04-dark-mode-design.md` for the rationale behind every color choice below — this plan's edits implement that spec exactly.
- Commit after each task with a `feat:` or equivalent conventional-commit message.

- [ ] **Step 3: Write the implementation**

Create `contexts/ThemeContext.tsx`:

```tsx
import React, { createContext, useContext, useState, useEffect, useCallback, FC, ReactNode } from 'react';

export type Theme = 'light' | 'dark';

interface ThemeContextValue {
  theme: Theme;
  toggleTheme: () => void;
}

const THEME_STORAGE_KEY = 'theme';

const ThemeContext = createContext<ThemeContextValue | undefined>(undefined);

function getInitialTheme(): Theme {
  const stored = localStorage.getItem(THEME_STORAGE_KEY);
  if (stored === 'light' || stored === 'dark') return stored;
  return window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light';
}

export const ThemeProvider: FC<{ children: ReactNode }> = ({ children }) => {
  const [theme, setTheme] = useState<Theme>(getInitialTheme);

  useEffect(() => {
    document.documentElement.classList.toggle('dark', theme === 'dark');
    localStorage.setItem(THEME_STORAGE_KEY, theme);
  }, [theme]);

  const toggleTheme = useCallback(() => {
    setTheme((prev) => (prev === 'dark' ? 'light' : 'dark'));
  }, []);

  return (
    <ThemeContext.Provider value={{ theme, toggleTheme }}>
      {children}
    </ThemeContext.Provider>
  );
};

export function useTheme(): ThemeContextValue {
  const ctx = useContext(ThemeContext);
  if (!ctx) {
    throw new Error('useTheme must be used within a ThemeProvider');
  }
  return ctx;
}
```
